# Implementation Plan - 全新系统设置与数据管护面板

## 🎯 Objective (目标)
为朔哥哥在形态选股前端 UI 界面中增设一个高能、清爽、可扩展的独立的 **“系统设置与数据仓库管护”Tab 页面**。支持对全市场 5500+ 只股票时序 K 线数据的实时管护、一键增量同步、可扩展的量化参数配置，以及极具极客质感的 **数据落库实时进度日志监视器**。

### 💡 核心安全保障：全自动自适应断点增量续传机制 (Resume-from-Break)
为了 100% 呼应并达成朔哥哥关于**“重启后端、或者数据同步中断后，能够接着上次更新完的断点继续跑，绝不重头拉取”**的重要安全诉求，系统数据中心网关的原生底层已具备了最高标准的**“毫秒级差分秒传与断点增量续跑”**物理设计：
1.  **最大日期自动回溯 (`max_date` 锚定)**：在对每只个股分发同步任务时，系统会自动去 `daily_bars` 检索该股在本地已落库的最大日期（例如该股已拉到 `2026-05-01`）。
2.  **动态设定开始日期**：
    *   **如果存在 `max_date`**：开始日期自动缩回至该日期，仅向腾讯/新浪备份接口请求该日期往后新增的增量日 K（耗时少于 0.05 秒，瞬间完成并跳至下一只股！）。
    *   **如果不存在 `max_date`（未拉过的股）**：才去从 `1990-01-01` 起拉全历史。
3.  **重启后极速断点续拉**：这意味着一旦后端重启、或者中断后重新按下同步，**之前已经拉过历史K线数据的上千只股票会因本地已存有 `max_date` 而在 0.01 秒内被自动识别并极速跳过（秒传）**！进度条会瞬间极速飙升、直奔中断时的那个“断点股票”继续向后吞吐！

*为了让朔哥哥在界面上倍感安心，我们还会在“系统设置”一键同步按钮下方，显式渲染一行生动的技术保证标签：*
`🛡️ 增量断点续传已激活：重启或中断后再次同步，系统会自动执行毫秒级秒传，自动跳过已更新标的。`

## 🏗️ Design Concept & Layout (界面与功能设计)

### 1. 独立 Tab 导航合并
*   将导航栏重构为 4 个 Tabs：
    1.  `scan` (📈 极速形态每日扫描大PK)
    2.  `backtest` (📊 历史形态滚动仿真回测)
    3.  `templates` (Sliders 形态模板与权重参数自进化)
    4.  `settings` (⚙️ 系统设置与数据仓库管护) `[NEW!]`

### 2. 系统设置页面结构设计 (Grid Layout)
系统设置页面采用 **分栏卡片式布局 (Double-Card Section)**，在保障极致科技深色质感的同时，为未来无缝撑开扩展骨架：

*   **左侧：股票数据仓库管护面板 (Data Warehouse Management)**
    *   **数据库状态仪表盘 (Live DB Stats Grid)**：
        展示由新增后端 `GET /api/stats` 接口反馈的真实落库指标：
        *   `基本面个股总数`: 展示当前已载入的股票数 (如 5528 只)。
        *   `K线时序行数`: 展示本地 `daily_bars` 表的总记录行数 (如 1,280,000 条)。
        *   `行情最新同步日期`: 展示当前时序数据中最大最新日期 (如 2026-07-19)。
    *   **行情流水线一键同步控制按钮组**：
        *   将原来位于 Header 右侧的“一键同步”按钮收纳并升级至此处。
        *   点击触发 `POST /api/jobs/sync-market-data` 后台异步同步任务。
    *   **同步流水线实时控制台 (Terminal-like Log Monitor)**：
        *   在下方配置一个类终端（等宽绿色字体、磨砂黑背景）的 **实时日志滚动监控器 (Live Sync Log console)**。
        *   轮询或动态载入并显示最新的同步状态日志，直观观测全市场行情高并发搬运的“均速xx股/秒”实时事实！

*   **右侧：扩展参数配置卡片 (Extensible Parameters Board)**
    *   *为系统未来的超参数微调设计表单输入区域，支持数据持久化存储至前端 `localStorage`：*
    *   `量化特征计算步长 (window_size)`: 默认 60 天。
    *   `并发拉取最高线程数 (max_workers)`: 默认 8 线程。
    *   `网络反爬延迟下限 (delay_min)`: 默认 100 毫秒。
    *   `网络反爬延迟上限 (delay_max)`: 默认 300 毫秒。
    *   `行情接口失败重试上限`: 默认 3 次。
    *   `反馈权重自进化学习率 (eta)`: 默认 0.05。
    *   提供 **`[ 保存系统全局配置 ]`** 按钮，点击一键持久化，且全系统所有 Tab 与网络层自动接入读取该配置，达成完美联动！

## 🛠️ Implementation Steps (开发步骤)

### Step 1: 后端支持新增统计接口 (GET /api/stats)
在 `backend/app/main.py` 注册并实现 `GET /api/stats`：
*   从 `stocks` 表中查询活跃股票总数。
*   从 `daily_bars` 表中查询行情总行数。
*   从 `daily_bars` 表中查询最大交易日期 `MAX(date)`。
*   通过 json 返回这些数据库实体事实。

### Step 2: 升级前端 App.tsx 路由与导航
*   在 `activeTab` 状态中追加 `'settings'` 分类。
*   在 Header 右侧移除旧的同步按钮，使 Header 极致清爽，将流量高聚类导流至设置页面中。
*   在 UI Tab 导航栏中，追加“⚙️ 系统设置与数据仓库管护”点击按钮。

### Step 3: 开发系统设置面板 settings UI
*   新增并编写 `renderSettingsTab` 渲染函数。
*   实现配置项超参数（`window_size`, `max_workers`, `retry_limit` 等）在 `localStorage` 中的存储 and 双向读取。
*   设计类 Terminal 的等宽滚动控制台，根据 `syncing` 状态定时拉取日志/进度事实展现。
*   编写精致的磨砂卡片、统计指标数显卡与表单滑块 CSS（融入 `App.css`）。
