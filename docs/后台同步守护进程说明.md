# PSE 后台行情同步守护进程

## 概述

sync_daemon.py 是一个完全独立于 FastAPI 主后端的后台数据同步进程。
它通过 Windows 计划任务在每个交易日收盘后（默认 16:00）自动执行增量同步，
确保主后端的日K数据始终新鲜，且主后端重启不影响同步任务。

## 核心设计

- **独立进程**：不依赖 FastAPI，作为单独 Python 进程运行
- **增量同步**：默认只拉取当天缺失的数据，首次全量后后续极快
- **PID 锁**：防止重复运行，僵尸进程自动清理
- **状态持久化**：PostgreSQL data_sync_status 表记录每次同步状态
- **独立日志**：logs/sync_daemon.log 按日滚动，最多保留 30 天
- **优雅退出**：支持 SIGINT/SIGTERM，通过世代 ID 机制优雅停止

## 快速开始

### 1. 首次全量同步（仅第一次需要）

`powershell
cd E:\IDEAProject\形态选股软件\PSE
python -m backend.app.data_center.sync_daemon --full
`

预计 10-30 分钟。

### 2. 安装每日自动同步（Windows 计划任务）

`powershell
python -m backend.app.data_center.sync_daemon --install-scheduler --scheduler-time 16:00
`

或直接双击 install_sync_scheduler.bat。

### 3. 查看同步状态

`powershell
python -m backend.app.data_center.sync_daemon --status
`

或 API 方式：

`powershell
curl http://127.0.0.1:8000/api/jobs/sync-status
`

### 4. 手动增量同步

`powershell
python -m backend.app.data_center.sync_daemon
`

### 5. 卸载计划任务

`powershell
python -m backend.app.data_center.sync_daemon --uninstall-scheduler
`

或直接双击 uninstall_sync_scheduler.bat。

## API 接口

新增端点：GET /api/jobs/sync-status

返回最近一次同步状态、耗时、以及数据库实际最新日K日期。

## 文件清单

- ackend/app/data_center/sync_daemon.py - 守护进程主脚本
- install_sync_scheduler.bat - 一键安装脚本
- uninstall_sync_scheduler.bat - 一键卸载脚本
- logs/sync_daemon.log - 守护进程日志（按日滚动）
- logs/sync_daemon.pid - PID 锁文件（运行时存在）

## 与原有同步的关系

- 原有 /api/jobs/sync-today-data 和 /api/jobs/sync-market-data 仍可正常使用
- 新增守护进程使用相同的 DataCenterSync 引擎，逻辑完全一致
- 两者共用数据库，互不冲突
- 前端可调用 /api/jobs/sync-status 判断数据是否过期
