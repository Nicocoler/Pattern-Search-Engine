# Pattern Search Engine (PSE) 逐步开发实施方案 (Detailed Implementation Plan)

Version: 1.0  
Date: 2026-07-19  
Author: Gemini CLI / 助手 (for 朔哥哥)

本方案旨在为 **Pattern Search Engine (PSE) - 基于 K 线形态相似搜索的智能选股系统** 制定一条从底层数据中心到前端可视化工作台的、极具实操性的渐进式开发路线。

由于 A 股市场存在特有的交易制度（如除权跳空失真、主创板块波幅不均、一字涨停板无流动性等），本方案在各个阶段均融入了针对性的实战容错与算法修正，确保系统在实战中表现出 **稳定、无未来函数、自适应对齐、100%可复现** 的四大特质。

## 🛠️ 各阶段详细开发方案设计

### Phase 1: 基础设施与数据中心 (Data Center) 建立

#### 1. 核心任务
打通安全、合规且带有高容错性的 A 股数据底座。解决 A 股收盘数据清洗滞后、分红派息除权跳空失真、网络高频反爬等核心痛点。

#### 2. 技术细节与方案
1. **TimescaleDB 超表与关系表设计**：
   * 创建 `stocks` 基本面信息表，记录代码、名称、上市日期、板块（主板/创业板/科创板/北交所）、ST 标记、是否停牌等状态。
   * 创建 `daily_bars` 时序表，字段包括：`(code, date, open, high, low, close, volume, amount, factor)`。使用 TimescaleDB 将该表转化为超表（Hypertable），按时间维度（`date`）进行分区。
   * 创建 `dirty_factors` 任务表 `(code, dirty_date, is_processed, updated_at)`，作为异步历史重算的触发事件缓冲池。
2. **AkShare 行情网关与反爬保护**：
   * 对接 `ak.stock_zh_a_spot_em`（用于更新每日板块名单、ST 标签和当前停牌状态）。
   * 对接 `ak.stock_zh_a_hist`（拉取 QFQ 前复权历史日 K 价格及成交量，必须携带复权因子 `factor`）。
   * 行情下载网关中实现 `Semaphore(5)` 并发限制器，并在每次请求之间植入 `100ms ~ 300ms` 的随机波动延时，配合指数退避重试（Exponential Backoff）算法规避源头风控。
3. **18:30 收盘哨兵网关保护机制**：
   * 考虑到 AkShare 等开源数据源在 15:00 收盘后清洗数据存在延迟，系统定时同步任务设定在晚间 18:30 触发。
   * **哨兵防护**：启动时，Data Center 首先查询 3 只代表性、高流动性的权重蓝筹（上证指数 `000001`、万科A `000002`、贵州茅台 `600519`）的当日日 K 行情。
   * 若 3 只哨兵的当日收盘价均非空且不为 0，视为全市场行情清洗完毕，开始全市场股票增量抓取；否则，执行指数级退避，延迟 15 分钟、30 分钟、60 分钟后再试，最大重试 5 次。
4. **除权差分核算与全历史重算触发器**：
   * 增量同步当日日 K 时，比对今日最新复权因子与昨日复权因子。
   * 一旦发现 `factor_today != factor_yesterday`（说明发生了除权除息），代表该股全历史的前复权价格均已改变。
   * 将该股代码和当前日期写入 `dirty_factors` 临时任务表。
   * 触发异步后台重算 Worker，清空并重新下载该股的全历史日 K，并强制重算其全历史指标与特征，彻底杜绝除权导致的“断崖式”假信号。

#### 3. 验收标准 (TDD)
- [x] 并发测试：稳定无阻碍地抓取全市场 5300+ 股的当天日 K。
- [x] 模拟除权测试：手动修改个股的前日 `factor`，触发同步时脚本能精准拦截，将个股写入 `dirty_factors`，并在后台异步重构该股的全部前复权历史价格。

### Phase 2: 基础指标引擎与特征插件 (Feature Engine)

#### 1. 核心任务
将原始的价格和成交量数据，转化为包含趋势、阻力、量能、实体、波动率在内的多维、无量纲特征矩阵。消除板块差异与极端行情（一字板）对形态特征匹配的严重干扰。

#### 2. 技术细节与方案
1. **基础指标引擎 (Indicator Engine)**：
   * 基于 Pandas/Polars 编写向量化指标计算包：MA5/10/20/60/120，BOLL 中轨/上轨/下轨及带宽（Width），RSI14，ATR14 波动率，以及 20日成交均量。
   * 极值容错：遇到停牌日，成交量输出 0，价格沿用上一交易日，严禁插入空值导致指标链断裂；上市不满 60 天的新股或数据不足时，统一输出 `None / NaN`。
2. **主创板块涨跌幅限制归一化系数 (Limit Normalization Factor, LNF)**：
   * **背景**：主板波动限制为 $\pm10\%$，创业板/科创板为 $\pm20\%$。如果不进行折算，创业板在趋势和波动率指标上天然具有更大的数值，会造成形态搜索倾向于高振幅个股或低估其相似度。
   * **计算公式**：引入修正系数：
     $$LNF = \frac{10.0}{LimitRatio}$$
     其中主板 $LimitRatio = 10$，创业板 $LimitRatio = 20$，北交所 $LimitRatio = 30$。
   * **特征乘数修正**：任何单日价格变动率、振幅、ATR、波动率等波动特征计入特征矩阵前，必须乘以 $LNF$，统一拉平折算为“主板等效波动率”。
3. **一字板极端形态插值修正 (One-Line Limit Board Interpolation)**：
   * **背景**：一字涨跌停板时，当日振幅为 0，且几乎无成交量。这会导致常规 K 线实体占比、上下影线比例公式的分母为 0 崩溃，或成交量突变破坏形态连贯性。
   * **处理**：特征插件若识别到一字板（当日最高价等于最低价，且当日收盘价处于涨跌停阈值内）：
     * K 线实体占比、影线占比、ATR 角度等特征直接**继承前一交易日**的有效值。
     * 成交量倍率（相对于20日均量）直接归零。
     * 在特征矩阵中追加标签字段 `is_limit_one_line: 1`。

#### 3. 验收标准 (TDD)
- [ ] 分类验证：选取主板和创业板走势结构一致的假数据，计算出的 LNF 修正特征距离接近于 0。
- [ ] 一字板容错：模拟连续 3 天一字涨停板的价格序列，特征生成无 `NaN` 或零除崩溃，实体和影线指标完美平滑过渡。

### Phase 3: 高斯模糊事件识别引擎 (Event Engine)

#### 1. 核心任务
用柔性的连续置信度概率代替冷冰冰的硬编码规则判断，把时序数据映射为一系列经典的“高斯置信度”物理事件。

#### 2. 技术细节与方案
1. **统一事件识别器接口 (EventDetector)**：
   * 实现统一基类，子类继承后负责识别单类事件。返回 `PatternEvent` 列表，每项包含：`event_type`、`start_date`、`end_date`、`confidence`（0.0 ~ 1.0 的连续概率值）及 `evidence`（测量证据）。
2. **高斯单峰正态置信度公式**：
   * 每个物理事件均配置一个实际物理偏离测量值 $x$、最优期望值 $\mu$ 以及偏离容忍标准差 $\sigma$。
   * 置信度（Confidence）计算公式：
     $$Confidence = \exp\left( -\frac{(x - \mu)^2}{2\sigma^2} \right)$$
   * 例如：回踩布林中轨事件 `TOUCH_BOLL_MIDDLE`：
     * 测量值 $x$ 即为 `(当日最低价 - 布林中轨) / 布林中轨`。
     * 最优期望 $\mu = 0$（完美回踩贴合）。
     * 设定偏离标准差 $\sigma = 0.015$。若该股最低价跌破或高出中轨 $1.5\%$，其高斯置信度为 $\exp(-0.5) \approx 60.6\%$；若完美贴合，置信度为 100%。
3. **内置八大 MVP 核心事件检测器**：
   * `TREND_UP`（均线呈多头角度且间距拉开）
   * `TOUCH_BOLL_UPPER`（最高价贴近或刺穿布林上轨）
   * `PULLBACK`（价格连续重心向下回调）
   * `TOUCH_BOLL_MIDDLE`（最低价回踩布林中轨）
   * `BOLL_MIDDLE_SUPPORT`（中轨依然维持向上斜率，且后续价格站稳中轨之上）
   * `VOLUME_SHRINK`（回调期间，成交量小于 20 日成交均量的 0.7 倍）
   * `STOP_FALLING_CANDLE`（收盘价收于当日振幅上半区，且伴随明显长下影线或窄实体）
   * `VOLUME_BREAKOUT`（单日成交量放大到 20 日均量的 1.8 倍以上，且收中阳线）

#### 3. 验收标准 (TDD)
- [ ] 边界衰减测试：输入一系列离布林中轨偏差由 0% 渐变到 5% 的数据，事件置信度输出值必须表现出完美的平滑钟形正态衰减，无硬性突跳。

### Phase 4: 多维 DTW 相似度与反馈自学习 (Similarity Engine)

#### 1. 核心任务
解决形态对齐中的“量纲冲突”（如高低价股绝对距离差）与“时间拉伸变形”（如洗盘时间长短不同），并通过用户的看盘好恶反馈在线微调模板权重，实现越用越聪明的个性化形态推荐。

#### 2. 技术细节与方案
1. **局部滑动窗口 Z-score 标准化**：
   * 在进行相似度对齐运算前，必须在当前的 60 日滑动窗口内，独立对每一个候选子序列特征维度计算均值和标准差并执行 Z-score 归一化：
     $$X'_{t, f} = \frac{X_{t, f} - \mu_{window, f}}{\sigma_{window, f}}$$
   * 严禁在全历史序列或全市场维度上做全局归一化，必须使用**局部 60 日滑动窗口归一化**，彻底抹平不同股票价格基数（如100元 vs 5元）和成交量量级的绝对差异。
2. **Sakoe-Chiba 限宽对角线约束多维加权 DTW**：
   * 引入带有 **Sakoe-Chiba 约束** 的多维加权 DTW 算法。转移对角线路径带宽 $W$ 限制为 $W = \text{floor}(0.15 \times \text{window\_size})$。对于 60 日窗口，转移路径严格限制在对角线两侧宽度为 9 的窄带内。
   * 此约束不仅将算法的时间复杂度从经典的 $O(N^2)$ 压低到 $O(N)$，而且限制了极端的时间扭曲变形（例如将模板中的 1 天强行对齐候选股中的 20 天）。
   * 多维欧氏对齐距离引入特征权重 $w_f$：
     $$D(i, j) = \sqrt{\sum_{f=1}^{F} w_f \cdot \left( T'_{i, f} - C'_{j, f} \right)^2}$$
3. **两级预过滤（匹配剪枝）**：
   * **一级硬过滤**：跳过停牌、ST股、日均成交额不足 1000 万的流动性僵尸股、及上市未满 60 天的新股（直接剪枝约 30% 冗余）。
   * **二级形态初筛**：快速核验候选股在最近 15 天内是否发生过 `TOUCH_BOLL_MIDDLE` (回踩中轨，且置信度 $> 30\%$) 的记录。若不满足，直接剔除（剪枝约 90% 冗余，使高耗时的多维 DTW 仅对最有希望的 10% 候选股执行）。
4. **在线反馈权重自更新**：
   * 提供 API 接收人工 Good/Bad 反馈。
   * 触发自适应梯度更新：对判定为 Good 的股票，拉高在局部对准中契合度高（距离小）的特征维度权重 $w_f$，降低距离大的特征维度权重；对于 Bad 反馈，执行相反动作。
   * **L1 归一化限制**：更新后的权重向量必须执行 L1 归一化：
     $$w_f = \frac{w_f}{\sum_{k=1}^{F} w_k}$$
     保持 $\sum w_f = 1.0$，确保权重不发生无限发散，自适应拟合朔哥哥的看盘直觉。

#### 3. 验收标准 (TDD)
- [ ] 性能红线：全市场 5300 股单窗口 60 日相似度匹配（双级剪枝 + 限宽对角线 DTW）在单机下运行耗时控制在 3 分钟以内。
- [ ] 权重收敛：进行 5 次加星/踩操作，打印模板维度权重 $w$，验证权重发生平滑偏转且其和恒等于 1.0。

### Phase 5: 形态无偏滚动回测系统 (Backtest Engine)

#### 1. 核心任务
为形态选股提供无偏的历史业绩支撑与胜率验证，必须在底层强制杜绝未来函数穿透。

#### 2. 技术细节与方案
1. **未来函数切片审计隔离器 (Look-ahead bias containment)**：
   * **机制**：在模拟历史任意交易日 $t$ 的匹配环境时，切片隔离器会强制截断数据库中 $t$ 日之后的所有行情、指标与特征数据。相似度计算引擎在 $t$ 日绝对接触不到 $t+1$ 日及之后的任何衍生行情。
   * **集成测试设计**：构造一个 60 日历史样本 $t_0 \sim t$，故意在 $t+1$ 日构造一个超额暴涨信号。传入截至 $t$ 日的数据给系统，确保计算出来的相似得分与最终推荐列表，绝不受到 $t+1$ 日暴涨数据的任何侧向影响（结果差异为 0）。
2. **滚动仿真交易模拟器 (Sliding Simulation)**：
   * 用户输入指定的回测历史跨度（如最近 3 年），仿真器以滑动步长（如 1 个交易日）从起点滚动到终点。
   * 在每个历史交易日 $t$：
     * 执行相似度扫描，筛选出满足相似度阈值（如 $> 85\%$）的前 N 只高分个股。
     * 模拟在时刻 $t$ 的收盘价买入，并在持有固定周期（5个交易日、10个交易日、20个交易日）后以当天收盘价强制平仓。
3. **回测多维绩效看板数据生成**：
   * 汇总统计：历史触发总信号数、持有 5d/10d/20d 的统计胜率、平均最大超额收益、平均最大回撤、盈亏比。
   * **Alpha 超额计算**：每次持股的累计收益均减去同期“沪深300指数”的基准涨跌幅，统计形态所带来的纯粹超额 Alpha，并输出形态投资组合的净值曲线数据。

#### 3. 验收标准
- [ ] 隔离测试：通过故意篡改 $t+1$ 数据的集成测试，验证未来函数隔离度为 100%。
- [ ] 回测效率：单形态模板执行 3 年历史日滑动回测，总耗时控制在 10 分钟以内。

### Phase 6: FastAPI 异步后端与任务队列

#### 1. 核心任务
封装系统的 RESTful 契约，支持高并发任务挂起。对于多维 DTW 扫描、大样本历史滚动回测等 CPU 密集型、高耗时任务，采用异步队列架构优雅解耦。

#### 2. 技术细节与方案
1. **接口契约设计 (RESTful API)**：
   * `POST /api/v1/templates` (新建抽象/历史形态模板，配置初始特征权重及高斯置信度参数)。
   * `POST /api/v1/scan` (提交对指定日期进行相似度扫描的任务。支持指定 `is_async: true` 立即返回 `task_id` 并挂载后台运行)。
   * `GET /api/v1/scan/status/{task_id}` (轮询获取扫描进度、剩余时长及已匹配的高分股票列表)。
   * `POST /api/v1/feedback` (接收人工 Good/Bad 标注，微调对应模板的特征权重)。
   * `POST /api/v1/backtest` (触发 3 年形态历史回测异步任务，返回回测进度与任务 ID)。
2. **Celery 异步分布式队列与 Redis 状态缓存**：
   * 引入 Celery 协程池作为后台长耗时扫描和回测的执行单元，Redis 作为消息中间件及进度状态后端。
   * 特征计算和 DTW 匹配任务按股票分片（Chore-partitioning），多 Worker 并行消费，进一步压低全市场搜索延迟。
3. **晚间 18:30 错峰自动化流水线**：
   * 集成 APScheduler 定时模块，每天晚间 18:30 自动拉起同步流水线。
   * 校验哨兵网关 -> 成功后触发 AkShare 增量行情同步 -> 行情校验落库 -> 自动重新计算脏复权股票全指标与特征 -> 对系统保存的所有激活态形态模板执行全市场相似度匹配扫描 -> 将今日 Top 20 推荐结果落库 -> 推送今日看盘分析报告。

#### 3. 验收标准
- [ ] 并发稳健性：前端同时发起形态回测与当前全市场扫描任务，后端响应敏捷，不阻塞，CPU 占用分发到 Celery 独立子进程中稳定运行。

### Phase 7: React 归一化研盘工作台 UI

#### 1. 核心任务
为朔哥哥提供直观、现代、富交互的极速研盘工作站。打破传统的低效表格，实现高价股与低价股的“无差别归零百分比重叠对比”与“高斯事件气泡标记”。

#### 2. 技术细节与方案
1. **百分比归一化 K 线重叠组件 (KlineCompareChart)**：
   * 基于 ECharts 或 Lightweight Charts 开发一个自定义图表组件。
   * **归零折算机制**：不直接使用价格作为纵轴。对于模板 K 线（长度 $L_1$）和候选 K 线（长度 $L_2$），强制将各自的第一天收盘价设为基准点 $0\%$，后续每一天的收盘价折算为相对于第一天的累计涨跌幅百分比：
     $$PricePercent_t = \frac{Close_t - Close_{day0}}{Close_{day0}} \times 100\%$$
   * 将两组百分比序列同屏、重合绘制，使得 1000 元的贵州茅台和 5 元的低价股能完全贴合并排比对形态的神似度。
2. **高斯模糊事件气泡标注 (Event Markers)**：
   * 在同屏重叠 K 线图上，根据 Event Engine 识别出的物理事件的时间范围、置信度，在对应的 K 线天数上方或下方悬挂可视化事件小气泡。
   * 朔哥哥在鼠标悬浮到对应气泡（例如第 40 天的 `[回踩中轨(92%)]`）时，能够浮窗展示物理证据（例如：`最低价偏离中轨: -0.4%`, `Boll中轨方向: 向上斜率8.2度`），将匹配逻辑100%白盒化、透明化。
3. **人机互动反馈与回测面板**：
   * 在匹配详情页旁边，配置一键加星（赞，促进 L1 特征权重正更新）和一键删除（踩，促进权重负更新）按钮。
   * 设立专门的形态回测绩效面板，同屏渲染 5d/10d/20d 的统计柱状图，以及形態投资组合净值曲线与同期沪深 300 累计净值曲线的渐变面积对比图，直观展现该形态超越大盘的 Alpha 获利能力。

#### 3. 验收标准
- [ ] 重合精度：打开任意高相似度匹配对的详情，模板 K 线与候选 K 线在百分比坐标轴上能完美对齐，鼠标滑过能同时呈现两者的具体价格与事件置信度细节。

## 🛡️ 最终系统交付红线

在每个阶段完成及向朔哥哥演示前，代码必须无条件通过以下验收红线：
1. **未来函数零泄露**：任意形态搜索和回测审计中，历史交易日 $t$ 的匹配结果绝不受 $t+1$ 日及之后任何行情或复权因子的侧向干扰。
2. **全市场同步 18:30 自动就绪**：若上证指数等 3 只哨兵个股当日收盘价仍未同步，系统必须保持指数退避状态，绝不抢拉空数据，也不让同步挂起死锁。
3. **计算效率小于 3 分钟**：全市场 5300 股经预过滤后送入多维加权 DTW 匹配，总耗时在单机 8 进程/协程下必须小于 3 分钟，以便随时响应即时扫描需求。
4. **权重 L1 恒定**：无论用户对形态模板提交多少次正负反馈，特征权重向量的累计和始终保持为 1.0，梯度微调绝不越界。
