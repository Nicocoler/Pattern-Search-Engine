# PSE Web 前端系统设计

Version: 1.1  
Date: 2026-07-18

## 1. 产品定位

前端不是营销页面，而是一个研究和筛选工作台。它应该安静、清晰、高信息密度，适合长期盯盘、对比和复盘。

核心目标：
- 快速查看每日 Top N。
- 理解每只股票为什么入选。
- 极度直观地多维对比模板与候选个股走势。
- 标注反馈，持续优化系统特征权重。

## 2. 页面结构

### 2.1 每日扫描结果

内容：
- 模板选择、扫描日期、数据就绪标记。
- Top N 表格、综合评分、分项评分、当前所处形态阶段、风险提示。

操作：
- 双击或点击“详情”跳转至对比详情页。
- 快捷标记“好匹配(good_match)”、“不像(bad_match)”、“加入观察(watchlist)”。
- 导出符合形态的个股 CSV/通达信代码导入包。

### 2.2 模板管理

内容：
- 模板列表（历史股票模板 / 抽象特征模板）。
- 活动状态、特征版本、权重矩阵、核心必需事件链。

操作：
- 新建历史模板、新建抽象形态。
- 编辑或由反馈自动微调后的特征权重实时修改。
- 试跑单股、归档模板。

### 2.3 股票详情多维对比页 (核心)

内容：
- 候选股票与模板的 **同屏 K 线拟合对比图**（重点解决股价绝对值无法对齐的痛点）。
- 对应 K 线上标记的 **可视化事件链气泡标签**。
- 分项评分雷达图、相似原因、差异细节。

### 2.4 反馈中心

内容：
- 历史已标注结果、正负反馈比例。
- 反馈对模板权重微调的影响轨迹。

## 3. 核心图表设计：同屏百分比归一化对比图

股价绝对值在 A 股相差巨大（如高价贵州茅台 1500 元 vs 低价工商银行 5 元）。若采用常规绝对坐标，多只个股形态绝无可能在同一个图表中重叠拟合。

### 3.1 基准百分比归一化 (Percentage-based Base Alignment)

在对比详情组件 `KlineCompareChart` 中，系统支持**百分比对齐渲染模式**：
1. **基准定义**：将模板起始日 $T_0$ 的收盘价和个股起始日 $C_0$ 的收盘价均定义为基准起点 `0%` (或基准数 `100`)。
2. **日波动折算**：之后的每一个交易日 $t$，其收盘价格 $P_t$、开盘价、最高价、最低价一律折算为相对于起点的百分比波动率：
   $$P_{pct, t} = \frac{P_t - P_0}{P_0} \times 100\%$$
3. **同屏绘制**：将模板和个股的两组百分比 K 线重叠在一个网格中。ECharts 中的主纵轴设为百分比 `[%]`，从而实现不同绝对价格的股票“骨架与走势曲线完全重合并拢”，一眼看穿形态贴合度。

### 3.2 辅助指标同屏折算

布林带上下轨、MA20 均线一律按照各自对应个股的起始价格进行百分比折算，使得对比图在拟合主价格的同时，其布林结构和趋势角度也能实现完美的视觉并排比对。

## 4. K 线图事件标注 (Event Markers Visualisation)

为提供极致的可解释性，详情 K 线图必须将高斯模糊识别到的事件节点标注在对应的 K 线柱上。

### 4.1 标注设计

使用轻量 K 线库 (Lightweight Charts) 的 `setMarkers` 接口或 ECharts 的 `markPoint` 接口：
* **TREND_UP**：在趋势段起始，K 线下方打上绿色向上小箭头（`shape: 'arrowUp'`）。
* **TOUCH_BOLL_MIDDLE**：在回踩中轨的那天，K 线正下方标注蓝色小圆形气泡，并注明文本 `中轨回踩`，提示框鼠标悬浮时展示其原始置信度（如 `置信度: 88.2%`）。
* **VOLUME_BREAKOUT**：在放量二次启动的 K 线正上方，标注红色闪烁小气泡，文本注明 `放量启动`。
* **效果**：形态对齐不仅仅是数学上的相似，通过可视化气泡，朔哥哥看盘时能直接看清系统的“思考过程”，增强对搜索结果的信任感。

## 5. 前端核心组件命名与状态规范

### 5.1 组件清单

* `KlineCompareChart`：支持百分比归一化同屏对比及事件气泡标注的复合 K 线图表。
* `RadarScoreChart`：展示趋势、布林、成交量、K线结构等分项得分的雷达/柱状图。
* `FeedbackActionGroup`：提供 quick action（好/坏/观察）的反馈按钮组，点击后异步发送 POST 触发权重自适应调整。
* `EvidenceExplanations`：根据 API 返回的 evidence 事实，分条渲染的正向/负向可解释原因列表。

### 5.2 扫描状态状态机 (Search Runs FSM)

前端通过 WebSocket 或轮询查询任务状态并转换 UI 徽章：
* `pending` (灰色，等待调度)。
* `running` (蓝色闪烁，执行全市场扫描中，展示当前计算进度，如 `已匹配 45%`)。
* `success` (绿色，执行成功，展示 Top N 结果)。
* `failed` (红色，执行失败，可点击查看 error 描述)。

## 6. 性能验收标准

- 图表在 60 日窗口多维数据下渲染延迟低于 50ms。
- 百分比归一化对齐切换流畅，两只股票 K 线和指标不产生变形或乱序。
- 详情对比页可以承载 20+ 个可视化气泡标签（Event Markers）而不产生卡顿。
- 提供符合深色主题（Dark Mode）的高品质 UI，适配量价及布林线在盯盘时的视力保护。
