# PSE 事件识别引擎设计

Version: 1.1  
Date: 2026-07-18

## 1. 模块定位

Event Engine 把连续多维特征转换为交易事件。系统不要求每个股票上涨和回调天数完全一致，而是判断**高置信度的事件顺序和时间关系是否保持一致**。

本系统摒弃传统量化软件“硬编码一刀切”的判定规则，引入**高斯模糊逻辑（Gaussian Fuzzy Logic）**，对所有事件输出平滑的连续置信度（Confidence, $0.0 \sim 1.0$），彻底解决边缘形态因微小误差而被硬性漏检的硬伤。

## 2. 核心数学模型：高斯置信度函数

对于任何连续特征测度 $x$（如价格与均线距离、缩量比例等），系统定义其事件触发的目标最优中心值 $\mu$ 与容错标准差 $\sigma$。事件置信度 $C \in [0, 1]$ 采用单峰正态分布函数计算：

$$Confidence(x; \mu, \sigma) = \exp\left( -\frac{(x - \mu)^2}{2\sigma^2} \right)$$

* **特征表现**：
  * 当特征值 $x$ 完美符合最优形态目标 $x = \mu$ 时，置信度为最高值 $1.0$。
  * 当特征值 $x$ 偏离最优目标，置信度按钟形曲线平滑衰减，哪怕在偏离边界上也不会产生断崖式跳跃（防止 0/1 边界抖动异常）。
  * 在事件识别结果中，保留时间范围、置信度以及原始证据。

```json
{
  "event_type": "TOUCH_BOLL_MIDDLE",
  "start_date": "2026-07-01",
  "end_date": "2026-07-03",
  "confidence": 0.882,
  "evidence": {
    "min_boll_middle_distance": 0.0075,
    "target_mu": 0.0,
    "tolerance_sigma": 0.015
  }
}
```

## 3. MVP 核心事件模糊判定规则设计

### 3.1 TREND_UP（趋势上涨）

* **特征测度**：MA5/MA10/MA20 多头排列天数占总窗口比例 $x$。
* **参数配置**：最优目标 $\mu = 1.0$，容差 $\sigma = 0.2$。
* **物理意义**：100% 的多头排列置信度为 $1.0$；若有 80% 的排列（存在几天短暂回调缠绕），置信度仍为 $0.95$；仅当多头排列低于 40% 时，置信度才衰减到 $0.1$ 以下。

### 3.2 TOUCH_BOLL_UPPER（触碰或贴近布林上轨）

* **特征测度**：窗口内 close 到上轨的最短距离比例 $x = (close - boll\_upper) / close$。
* **参数配置**：最优目标 $\mu = 0.0$（贴合上轨），容差 $\sigma = 0.015$（即容许在 1.5% 范围内摆动）。
* **物理意义**：直接触碰置信度为 $1.0$；距离 1.5% 置信度为 $0.61$；距离 3%（未完全碰到，仅贴近）置信度仍有 $0.135$（不会被一刀切丢弃）。

### 3.3 PULLBACK（上涨后回调）

* **特征测度**：窗口内最高点回调幅度百分比 $x$。
* **参数配置**：模板定义的合理回调幅度 $\mu = 0.10$（回调 10% 最佳），容差 $\sigma = 0.05$（5% 容差）。
* **物理意义**：回调 10% 最佳；若回调 5% 或 15%，置信度仍处于 $0.6$ 以上；若回调超过 25%，则判定为破位，置信度几乎归零。

### 3.4 VOLUME_SHRINK（回调缩量）

* **特征测度**：回调阶段平均成交量倍率 $x = volume / volume\_ma20$。
* **参数配置**：最优目标 $\mu = 0.50$（缩量至均量的一半），容差 $\sigma = 0.25$。
* **物理意义**：量能萎缩在 0.5 倍左右时，置信度达到最高；若不缩量反放量（$x \ge 1.0$），置信度直接跌入极低区间（低于 $0.1$）。

### 3.5 TOUCH_BOLL_MIDDLE（回踩布林中轨）

* **特征测度**：单日低点或收盘到中轨的距离比值 $x = (low - boll\_mid) / boll\_mid$。
* **参数配置**：最优目标 $\mu = 0.0$（精确踩线），容差 $\sigma = 0.015$。
* **物理意义**：正好回踩中轨置信度为 $1.0$；偏离 1.5% 置信度为 $0.61$；偏离 3%（即并未精准回踩，离中轨还有一定间距就拉起）置信度为 $0.135$。

### 3.6 BOLL_MIDDLE_SUPPORT（中轨支撑成立）

* **特征测度**：回踩中轨后，close 没有跌破中轨的天数占比 $x$。
* **参数配置**：最优目标 $\mu = 1.0$，容差 $\sigma = 0.15$。
* **物理意义**：回踩后始终守住中轨置信度为 $1.0$；若有 1 天盘中瞬间击穿但迅速收回，置信度平滑微降。

### 3.7 STOP_FALLING_CANDLE（出现止跌 K 线）

* **特征测度**：K 线收盘价在全天振幅中的位置百分比 $x = (close - low) / (high - low)$。
* **参数配置**：最优目标 $\mu = 0.80$（长下影线或小实体阳线，收盘在高位），容差 $\sigma = 0.15$。
* **物理意义**：收盘在振幅上四分之一区间，置信度高；收在低点则无止跌迹象。

### 3.8 VOLUME_BREAKOUT（放量二次启动）

* **特征测度**：当日成交量倍率 $x = volume / volume\_ma20$。
* **参数配置**：最优目标 $\mu = 2.0$（倍量启动），容差 $\sigma = 0.5$。
* **物理意义**：倍量（2.0倍）或以上置信度极高；若缩量拉升（如成交量仅 0.8 倍），置信度迅速滑落。

## 4. 事件序列柔性匹配算法（Elastic Event Sequence）

在布林回踩中轨二次启动模板匹配中：

```text
TREND_UP
  -> TOUCH_BOLL_UPPER
  -> PULLBACK
  -> VOLUME_SHRINK
  -> TOUCH_BOLL_MIDDLE
  -> BOLL_MIDDLE_SUPPORT
  -> STOP_FALLING_CANDLE
  -> VOLUME_BREAKOUT
```

### 4.1 序列加权合并公式

事件序列得分不只依赖事件存在性，而是与其识别置信度高度绑定。在对齐后的事件序列，其序列得分计算如下：

$$Score_{sequence} = \sum_{e \in Required} \beta_e \cdot Confidence_e - \sum_{pe \in AntiEvents} Penalty_{pe}$$

* $\beta_e$：每个必需事件在链条中的相对权重（如 `TOUCH_BOLL_MIDDLE` 占 0.25，`VOLUME_BREAKOUT` 占 0.2）。
* $Confidence_e$：高斯模糊函数输出的该事件置信度。
* $Penalty_{pe}$：反向违规事件惩罚。例如若在回踩期间，检测到 `CLOSE_BREAK_BOLL_LOWER` (收盘跌破下轨，置信度 $0.8$)，则扣减大额分数。

## 6. 验收标准

- 所有核心事件摒弃一刀切的硬阈值过滤，全部升级为单峰高斯置信度平滑曲线公式。
- 支持定义最优期望均值 $\mu$ 与偏离容忍标准差 $\sigma$，并在 evidence 中保留。
- 反向破位事件（如跌破下轨）作为置信度扣分项。
- 事件序列的相似度由高斯置信度加权求和，能够对实盘中的微调震荡形态实现高抗噪。
