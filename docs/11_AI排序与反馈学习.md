# PSE AI 排序与反馈学习设计

Version: 1.1  
Date: 2026-07-18

## 1. 模块定位

Ranking Engine 负责把相似度引擎算出的多维分项评分转成最终呈现给用户的最终排序（Rank No）。它不仅要提供高解释性的排序，还承载着形态搜索的核心自律进化闭环：通过接收人工反馈，自适应地调整形态模板特征权重。

## 2. 第一阶段：在线自适应规则加权排序

第一阶段主要采用规则加权和风险惩罚，但在用户标注后，**模板权重会自动进化**，实现不用重新训练 AI 模型也能“越用越懂用户”的选股体验。

### 2.1 基础排序公式

$$\text{TotalScore} = \sum_{f \in F} w_f \cdot S_f - \text{RiskPenalties}$$

* $S_f$：分项相似度评分（趋势、布林、成交量、K线、事件、波动率，范围 $0 \sim 100$）。
* $w_f$：特征维度权重，且约束 $\sum_{f \in F} w_f = 1.0$。
* $\text{RiskPenalties}$：形态破位或反向事件硬性罚分。

## 3. 在线自适应反馈算法（Adaptive Online Weight Adjustment）

为了让系统能够自适应捕捉用户看盘时的“直观盘感”（例如，有些用户对“成交量节奏”更挑剔，有些用户对“布林回踩深度”更敏感），第一阶段专门引入了基于反馈的自适应迭代。

用户可在结果页对个股标注以下标签：
* `good_match`（正反馈：形态非常符合预想）。
* `bad_match`（负反馈：不像，判定为干扰）。

### 3.1 正反馈权重强化逻辑

当用户将某只股票（相似度向量为 $S = [s_1, s_2, ..., s_F]$）标记为 `good_match` 时，如果该股总分偏低，说明用户欣赏它里面得分较高的某些特征。
系统对模板的特征权重执行自适应微幅上调：

$$w_f \leftarrow w_f + \eta \cdot \left( S_f - \bar{S} \right) \cdot w_f \cdot (1 - w_f)$$

* $\bar{S}$：该个股在所有分项评分上的平均值。
* $\eta = 0.05$：在线学习率（Learning Rate）。
* $(S_f - \bar{S})$：若该特征评分显著高于平均水平，获得更多权重加成。
* $w_f(1-w_f)$：非线性衰减项，防止权重越界到 0 或 1 之外，且保证权重调整平滑。

### 3.2 负反馈权重弱化逻辑

当用户将高分股标记为 `bad_match` 时，说明用户认为“该形态看起来不像”。系统对表现过高、导致用户反感的分项（例如布林很像但成交量完全对不上）进行权重惩罚：

$$w_f \leftarrow w_f - \eta \cdot \left( S_f - \bar{S} \right) \cdot w_f \cdot (1 - w_f)$$

### 3.3 权重归一化

每次迭代更新后，系统自动对特征权重向量进行 L1 归一化，重新维持权重和为 1.0：

$$w_f \leftarrow \frac{w_f}{\sum_{k \in F} w_k}$$

这一机制能让每个抽象模板在用户的持续标注中自适应进化，成为贴合朔哥哥人脑盘感的“私人订制”模板。

## 4. 风险惩罚项 (Risk Penalties)

在排序阶段，结合个股的异常信息进行硬性得分惩罚：
* **MA60/MA120 破位（中期空头趋势）**：扣减 20 分。
* **跌破布林下轨**：扣减 15 分。
* **一字板畸变标记 `is_limit_one_line == 1`**：对于成交量分项权重豁免，但如果属于跌停一字板，直接降权排序至末尾。
* **流动性缺失（日均成交额不足 500w）**：扣减 25 分。

## 5. 解释生成事实结构

解释生成机制必须建立在个股的 evidence 基础之上，严禁没有数据依据的虚假表述。

```json
{
  "explanation_facts": {
    "positive_facts": [
      {"field": "boll_middle_distance", "text": "价格精准踩线中轨，距离仅为 0.75%", "confidence": 0.88},
      {"field": "volume_ratio_20", "text": "回踩期间成交量萎缩至 20 日均量的 0.72 倍，浮筹清洗干净", "confidence": 0.91}
    ],
    "negative_facts": [
      {"field": "volume_ratio", "text": "二次启动日成交量仅为 1.2 倍，放量力度偏弱", "confidence": 0.55}
    ]
  }
}
```

## 6. 第二阶段：基于排序模型的学习 (Learning to Rank)

当标注库中的反馈数据累积超过 1000 条后，过渡到第二阶段：
* **数据收集**：以相似度向量、事件特征、行业分布作为输入特征，反馈 label 折算为目标（`good_match` = 2, `watchlist` = 1, `bad_match` = 0）。
* **排序模型**：采用 **LightGBM Ranker (Lambdarank)**，优化形态列表的相关性。

## 7. 第三阶段：基于表象的向量化 Embedding 表示

将 60 日局部特征矩阵，通过一个一维时序卷积神经网络（1D-CNN）或者 Transformer Encoder，映射为一个 128 维的密集特征向量：
* **极速检索**：利用 FAISS 或 PostgreSQL 的 `pgvector` 插件进行 HNSW 向量索引。
* **形态聚类**：自动对全市场形态进行无监督分类，给用户推荐未定义过的新形态。

## 8. 评估指标

* **Precision@10 / Precision@20**：每日推荐的 Top 10 或 Top 20 结果中，用户标记为 `good_match` 或 `watchlist` 的比例，目标第一阶段不低于 75%。
* **反馈转化率**：每日有结果输出后，用户主动进行反馈标注的占比。
* **超额收益率（Alpha）**：对 Top 10 个股后续 5-10 日进行超额跟踪，评估好匹配形态是否具备正向超额表现。

## 9. 验收标准

- 第一阶段排序流程完全可解释。
- 反馈标签（good/bad/watchlist/ignore）能通过 API 稳定入库。
- 引入正负反馈在线权重自适应调整公式，每次提交后模板的 $w_f$ 产生相应的自适应调整，且权重和始终为 1.0。
- 排序结果具有防跌停、防停牌、防破位的硬性过滤。
