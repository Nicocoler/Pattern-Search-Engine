# PSE 部署与运维方案

Version: 1.0  
Date: 2026-07-18

## 1. 部署目标

第一阶段支持本地单机运行，后续可平滑升级到服务器部署。

## 2. 本地开发环境

组件：

- backend FastAPI。
- frontend React。
- PostgreSQL。
- Redis。

推荐使用 docker-compose 管理数据库和缓存，本地直接运行后端与前端。

## 3. Docker Compose 服务

服务：

```text
postgres
redis
backend
frontend
worker
```

环境变量：

```text
DATABASE_URL
REDIS_URL
MARKET_DATA_PROVIDER
FEATURE_VERSION
ALGORITHM_VERSION
LOG_LEVEL
```

## 4. 任务调度

每日任务：

```text
15:10 同步行情
15:30 计算指标
15:45 生成特征
16:00 识别事件
16:15 执行模板扫描
16:30 生成报告
```

实际时间应根据数据源更新时间调整。

## 5. 日志

日志必须包含：

- request_id。
- job_id。
- run_id。
- symbol。
- template_id。
- duration_ms。
- status。
- error_message。

## 6. 监控

关键指标：

- 行情同步成功率。
- 指标计算耗时。
- 特征生成耗时。
- 扫描耗时。
- API 错误率。
- 数据库连接数。

## 7. 备份

开发环境：

- 每日 schema 备份。
- 每周样本数据备份。

生产环境：

- 每日全量备份。
- WAL 增量备份。
- 重要模板和反馈单独导出。

## 8. 安全

- API 鉴权。
- 数据源 token 不提交到 Git。
- `.env` 不入库。
- 用户反馈与配置操作保留审计日志。

## 9. 运维命令

建议提供：

```text
make dev
make test
make migrate
make sync-data
make scan TEMPLATE_ID=xxx
make backup
```

Windows 环境可额外提供 PowerShell 脚本。

## 10. 验收标准

- 新机器可根据 README 在 30 分钟内启动。
- 后端、前端、数据库、Redis 均有健康检查。
- 定时任务失败可重试。
- 日志能定位到具体 symbol 和任务阶段。
