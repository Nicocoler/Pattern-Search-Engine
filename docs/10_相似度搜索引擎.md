# PSE 相似度搜索引擎设计

Version: 1.1  
Date: 2026-07-18

## 1. 模块定位

Similarity Engine 负责判断候选走势与模板走势“像不像”。它是 PSE 的运算核心，采用两级预过滤和多维加权时间弹性对齐（Z-score + 多维 DTW + 事件匹配）的混合相似度算法。

## 2. 输入输出

输入：
- 模板特征矩阵（Template Feature Matrix，$M \times F$）。
- 候选股票特征矩阵（Candidate Feature Matrix，$N \times F$）。
- 模板及候选事件序列（Event Sequences）。
- 相似度权重及阈值配置（Scoring Config）。

输出：
- 综合相似度评分（Total Similarity Score，0~100 分）。
- 分项评分拆解（`score_breakdown`：趋势、布林、成交量、K线、事件、波动率）。
- 时间对齐路径及 matched_events。
- 解释性事实（`positive_facts` & `negative_facts`）。

## 3. 两级候选过滤剪枝（Pre-filtering Gates）

为攻克全市场 5300+ 股票的计算压力，必须在执行高耗能的 DTW 计算前过滤掉 90% 的无效标的。

### 3.1 第一级：流动性与板块硬过滤 (Hard Filters)

直接在 SQL 查询层或数据缓存层进行硬剔除：
* 停牌股剔除。
* 剔除成交额过低的“僵尸股”（例如：最近 20 日日均成交额低于 1000 万人民币）。
* 退市整理期、ST/*ST 股票（除非模板配置中明确允许）。
* 上市未满 60 个交易日的新股。

### 3.2 第二级：形态特征快速初筛 (Soft-Filtering Heuristics)

对于“回踩布林中轨二次启动”等形态，要求在个股近 15 个交易日内，至少满足以下轻量级指标特征之一：
* 触发了事件：`TOUCH_BOLL_MIDDLE` (回踩中轨)。
* 或者是布林带最小距离特征：`min(boll_middle_distance) <= 0.03`。
* 剔除：若近 15 天中轨斜率严重向下且从未贴近过中轨，则直接剪枝。
* **效果**：经过两级过滤，仅有 300~500 只个股进入下一阶段的多维 DTW 计算，直接缩减了 90% 的无效运算。

## 4. 多维特征标准化与量纲冲突消除

由于多维特征（价格、均线角度、成交量倍率、布林带宽等）的数值范围存在几个数量级的差异（例如距离在 `[-0.1, 0.1]` 之间，成交量倍率在 `[0.2, 10.0]` 之间），如果不进行局部标准化，DTW 计算时高数值的特征（如成交量）会完全淹没低数值特征，导致对齐路径完全失真。

### 4.1 窗口内 Z-score 归一化 (Local Window Standardisation)

对送入 DTW 计算的特征矩阵 $X \in \mathbb{R}^{W \times F}$（$W$ 为窗口天数，$F$ 为特征维度数），每一个维度 $f \in [1, F]$ **在其局部窗口序列内独立执行 Z-score 变换**：

$$X'_{t, f} = \frac{X_{t, f} - \mu_f}{\sigma_f + \epsilon}$$

其中：
* $\mu_f$：该特征 $f$ 在当前窗口 $W$ 内的均值（Mean）。
* $\sigma_f$：该特征 $f$ 在当前窗口 $W$ 内的标准差（Standard Deviation）。
* $\epsilon = 1e-6$：微小的常数，用于防止标准差为 0（如一字板振幅为 0）时产生除零异常。

### 4.2 特征对齐权重矩阵 (Feature Distance Weighting)

在计算多维 DTW 的某一步时间对齐距离时，局部点 $(i, j)$ 的欧几里得距离采用加权平方和：

$$D(i, j) = \sqrt{\sum_{f=1}^{F} w_f \cdot \left( T'_{i, f} - C'_{j, f} \right)^2}$$

其中 $T'$ 与 $C'$ 分别为标准化后的模板特征和候选特征， $w_f$ 为该特征维度的权重配置（$\sum w_f = 1.0$）。例如：
* 价格归一化特征：$w_{close\_norm} = 0.35$。
* 布林中轨距离特征：$w_{boll\_dist} = 0.25$。
* 成交量倍率特征：$w_{vol\_ratio} = 0.15$。
* 其它辅助结构特征：共占 0.25。

## 5. DTW 性能瓶颈加速（Performance Acceleration）

### 5.1 Sakoe-Chiba 约束窗口（Sakoe-Chiba Band）

为了防止 DTW 路径产生病态的对齐（例如模板的第 1 天对齐到候选的第 40 天），同时极大地减少计算量，我们必须对 DTW 的网格路径搜索施加**限宽约束**。

* **约束带宽度**：
  $$W = \max(3, \text{floor}(\delta \cdot \text{window\_size}))$$
  默认 $\delta = 0.15$。对于 60 日的形态匹配，对角线限宽 $W = 9$。
* **对齐条件**：仅在 $|i - j| \le W$ 的斜带内进行状态转移。
* **复杂度优化**：将时间与空间复杂度从 $O(N^2)$ 降低到 $O(N \cdot W)$，计算耗时缩减 80% 以上。

### 5.2 高性能加速库选型

* 严禁采用纯 Python 双重循环实现 DTW 算法。
* **推荐**：使用基于 C 底层优化的开源库 **`dtaidistance`** 进行计算；或者通过 **`Numba`** 的 `@jit(nopython=True, cache=True)` 将双重循环编译为高效率的机器码。

## 6. 综合评分与扣分机制（Scoring & Penalties）

相似度引擎通过多轴加权生成综合评分（Total Score，0~100 分）：

```text
total_score =
  0.25 * trend_score
+ 0.25 * boll_score
+ 0.15 * volume_score
+ 0.15 * candle_score
+ 0.15 * event_sequence_score
+ 0.05 * volatility_score
- risk_penalties (风险惩罚项)
```

### 6.1 风险硬性扣分项 (Risk Penalties)

若候选走势出现以下反向行为，给予硬性扣分：
* **破位惩罚**：最近 3 日内个股有效跌破布林下轨，或收盘价跌破 MA60，直接扣减 20 分。
* **节奏缺失惩罚**：回调阶段出现大阴线放量破位，或二次启动时未见放量（成交量倍率低于 1.0），扣减 10-15 分。
* **停牌惩罚**：若 60 日窗口内包含超过 3 日停牌，视为由于停牌扭曲走势，扣减 5 分。

## 7. 输出结构

扫描生成的每个候选结果需要输出详细、可解释的结构，以便在 Web 详情页还原其对齐形态：

```json
  }

```

## 8. 验收标准

- DTW 在计算前必须执行 60 日局部 Z-score 标准化，彻底消除量纲冲突。
- 必须支持特征距离权重配置，避免成交量过多干扰价格趋势对齐。
- 全市场 5300 股在有两级剪枝网关、Sakoe-Chiba 带宽（W=9）和 C 底层编译加速支持下，單机扫描在 3 分钟以内产出 Top 50 结果。
- 评分模型包含风险硬性扣分，保证高分股票形态的安全稳健性。
