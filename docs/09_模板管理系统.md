# PSE 模板管理系统设计

Version: 1.1  
Date: 2026-07-18

## 1. 模块定位

Template Engine 管理用户认可的形态模板。形态模板绝不仅仅是一个简单的 K 线静态图片，而是**由滑动窗口大小、特征筛选插件、高斯模糊事件、相似评分权重、流动性硬过滤、以及回测默认持股参数组成的结构化、参数化配置**。

## 2. 模板类型

### 2.1 历史模板 (Historical Template)

来自某只真实股票在历史上的某段走势，系统自动从中抽取出 60 日滑动窗口的多维特征矩阵与事件置信度链。

```json
{
  "template_type": "historical",
  "source_symbol": "600519",
  "start_date": "2026-03-01",
  "end_date": "2026-05-31",
  "window_size": 60
}
```

### 2.2 抽象模板 (Abstract Template)

直接由用户手动配置或从专家看盘经验中抽象出来的标准事件逻辑链条。

```json
{
  "template_type": "abstract",
  "event_schema": [
    "TREND_UP",
    "TOUCH_BOLL_UPPER",
    "PULLBACK",
    "VOLUME_SHRINK",
    "TOUCH_BOLL_MIDDLE",
    "BOLL_MIDDLE_SUPPORT",
    "VOLUME_BREAKOUT"
  ]
}
```

## 3. 模板详细配置结构

为确保模板既能支撑前台的**每日相似扫描**，又能完美供给后台的**历史无偏回测**，模板配置需要高度结构化：

```json
{
  "template_id": "uuid",
  "name": "布林回踩中轨二次启动",
  "window_size": 60,
  "status": "active",
  "version": "1.1.0",
  "hard_filters": {
    "min_amount_20d": 10000000,    // 最近20日日均成交额门槛 (1000w)
    "ma20_slope_min": 0.0,         // 中轨斜率最低限（防单边下行股）
    "allow_st": false,             // 是否过滤 ST/*ST 股
    "max_suspended_days": 3        // 60日内允许最大停牌交易日
  },
  "scoring_weights": {
    "trend_score": 0.25,
    "boll_score": 0.25,
    "volume_score": 0.15,
    "candle_score": 0.15,
    "event_sequence_score": 0.15,
    "volatility_score": 0.05
  },
  "event_schema": [
    {"event_type": "TREND_UP", "required": true, "weight": 1.0, "tolerance_sigma": 0.20},
    {"event_type": "TOUCH_BOLL_UPPER", "required": true, "weight": 1.0, "tolerance_sigma": 0.015},
    {"event_type": "TOUCH_BOLL_MIDDLE", "required": true, "weight": 1.2, "tolerance_sigma": 0.015},
    {"event_type": "VOLUME_BREAKOUT", "required": false, "weight": 0.8, "tolerance_sigma": 0.50}
  ],
  "default_backtest_config": {     // 专门提供给第 16 章回测系统的默认持仓参数
    "holding_periods": [5, 10, 20],
    "benchmark": "000300.SH",
    "score_threshold": 80.0
  }
}
```

## 4. 模板生命周期

模板的状态机转换定义如下：
* `draft` (草稿)：创建中，未经验证。
* `active` (可用)：已在 10 只样本股票中完成“试跑(Dry-run)”，评分与事件对齐符合逻辑，正式用于全市场每日扫描和回测。
* `archived` (归档)：废弃的历史模板，停止每日扫描，但数据库保留配置，防止历史扫描结果无法复现。
* `experimental` (实验性)：由自适应反馈更新或权重 SGD 调整出来的演进版本，与主版本并存。

```text
创建模板(Draft) 
  -> 样本试跑/回测验证 
  -> 校验成功 
  -> 激活(Active) 
  -> 每日扫描/反馈在线微调(L1自更新) 
  -> 演进新版本(Experimental) 
  -> 确认升级 
  -> 旧版本归档(Archived)
```

## 5. 模板版本管理

任何对以下核心属性的修改：
1. 必需事件链（`event_schema`）。
2. 初始权重矩阵（`scoring_weights`）。
3. 滑动窗口大小。
4. 依赖的底层 `feature_version` 升级。

**一律禁止执行 SQL UPDATE 覆盖原配置**。系统必须生成一个新的递增版本号（例如由 `1.1.0` 升级为 `1.2.0`），新建一行记录写入 `pattern_templates` 表，从而保证所有的历史扫描结果及任务完全符合“可审计性”与“100%可复现”。

## 6. 验收标准

- 能流畅创建、查询、激活和归档历史模板与抽象形态配置。
- 模板 schema 包含高斯模糊判定参数 `tolerance_sigma`（容错标准差）。
- 模板 schema 包含第 16 章回测引擎所需的 `default_backtest_config` 默认参数，打通回测契约。
- 扫描和回测结果必须外键关联 `template_id` 及其 `version` 字段，杜绝悬挂数据。
