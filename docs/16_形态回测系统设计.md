# PSE 形态回测系统设计

Version: 1.0  
Date: 2026-07-18

## 1. 模块定位

Backtest Engine（形态回测引擎）是 PSE 选股形态走向实战的核心科研层。它的职责是解决用户的终极痛点：**“我找出来的这种形态模板，在历史上买入的赚钱概率（胜率）究竟有多少？未来拿多少天卖出性价比最高？”**

通过将形态匹配（DTW+高斯事件）在历史时间轴上进行滚动重演，计算出该形态的历史统计胜率、盈亏比、最大回撤，以及相对于沪深 300 的超额收益（Alpha），避免纸上谈兵。

---

## 2. 回测核心参数模型

执行一次形态回测需要定义以下参数输入：

```json
{
  "template_id": "uuid",          // 需要回测的形态模板 ID
  "start_date": "2021-01-01",     // 回测历史起点
  "end_date": "2025-12-31",       // 回测历史终点
  "score_threshold": 80.0,        // 判定买入信号的评分门槛（如综合分 > 80分）
  "holding_periods": [5, 10, 20], // 虚拟持股天数周期（交易日）
  "max_portfolio_size": 10,       // 每日最大持股上限（若信号多，优先买评分最高的Top N）
  "benchmark": "000300.SH"        // 对比基准（如沪深 300 业绩指数）
}
```

## 3. 回测仿真流程与算法

回测采用 **每日滑动窗口重演仿真（Sliding Window Historical Simulation）**。

### 3.1 核心流程

```text
1. 初始化账户资金、基准指数历史收益率序列、空持股队列。
2. 提取 [start_date - 60日, end_date] 期间的全市场历史日 K、指标与特征数据。
3. 沿着交易日时间轴 t = start_date 到 end_date 循环：
     a. 特征锁定：提取 t 日往前推 60 交易日的“历史局部特征矩阵”。
     b. 防未来函数审计：确保特征提取、事件检测和评分算法仅使用截至 t 日的数据，绝对不泄露 t+1 日及之后的任何行情。
     c. 相似度计算：在全市场个股中，计算截至 t 日的 60 日窗口与模板的相似度评分。
     d. 信号生成：筛选出评分 > score_threshold 的个股，作为 t 日的“形态买入信号”。
     e. 虚拟交易执行：
        - 模拟在第 t+1 日以开盘价（或 t 日收盘价）买入。
        - 记录买入成本价 $P_{buy} = Close_{t}$ 或 $Open_{t+1}$。
     f. 持仓跟踪与出场：
        - 对买入的个股，跟踪并在第 t+5、t+10、t+20 交易日模拟按收盘价卖出。
        - 计算卖出收益率 $R = (P_{sell} - P_{buy}) / P_{buy}$。
     g. 超额计算：扣除同期基准指数（如沪深300）的涨跌幅，折算为超额收益（Alpha）。
4. 汇总全历史区间的虚拟交易明细，计算整体绩效。
```

## 4. 核心绩效指标（Performance Metrics）

回测引擎在回测结束后输出以下统计指标：

### 4.1 形态胜率 (Winning Rate)

对于设定的持股周期 $T \in [5, 10, 20]$：

$$\text{WinningRate}_T = \frac{\text{历史卖出收益率} R > 0 \text{ 的信号次数}}{\text{总触发买入信号次数}} \times 100\%$$

### 4.2 平均期望收益 (Average Return) 与 Alpha

* **平均绝对收益**：所有触发信号个股在持股 $T$ 日后的收益率平均值。
* **平均超额收益 (Alpha)**：
  $$\text{AverageAlpha}_T = \frac{1}{M} \sum_{i=1}^{M} \left( R_{stock, i} - R_{benchmark, i} \right)$$

### 4.3 风险与资金指标

* **最大回撤 (Max Drawdown)**：模拟净值曲线历史上的最大峰值回落幅度。
* **盈亏比 (Profit-Loss Ratio)**：所有盈利交易的平均盈利额 / 所有亏损交易的平均亏损额。
* **信号频次分布**：按年份、月份统计信号触发次数，评估形态在不同市场牛熊阶段（如单边熊市、震荡市、牛市）的适应度。

## 5. 严格审计：防未来函数（Look-ahead Bias Prevention）

在量化回测中，“未来函数”是导致回测胜率 90% 但实盘巨亏的罪魁祸首。
系统实施以下**未来函数硬性审计红线**：
1. **指标计算隔离**：在计算 $t$ 日的技术指标（如 BOLL、MA20）时，其 DataFrame 长度和范围必须被截断至 $t$ 日，绝对禁止引入 $t$ 日之后的收盘价。
2. **事件检测隔离**：事件识别引擎（Event Engine）在检测历史事件链时，其输入矩阵的最大索引必须是 $t$。
3. **除权因子一致性**：回测时必须使用当时已除权完毕的历史价格。由于 Data Center 在每日收盘后均比对除权差分并重新覆写了 `daily_bars` 历史，本系统的前复权日 K 本身是无偏、安全的，但回测时必须采用回测时间终点 $end\_date$ 时已知的最新前复权数据。

## 7. 验收标准

- 回测引擎能自动遍历历史交易日轴，并以日为单位滚动提取特征并进行全市场匹配。
- 指标计算、事件判定及相似度打分绝不含有未来函数。
- 能够准确输出 5d、10d、20d 等周期的平均胜率、绝对收益、超额收益及盈亏比。
- 能够输出每日虚拟账户的累计净值曲线数据，支持前端与基准指数曲线重叠对比。
