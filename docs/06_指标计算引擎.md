# PSE 指标计算引擎设计

Version: 1.0  
Date: 2026-07-18

## 1. 模块定位

Indicator Engine 负责把行情数据转换为基础技术指标。它只计算客观指标，不判断形态，也不参与最终评分。

## 2. 输入输出

输入：

- `daily_bars`
- adjusted close
- volume
- amount

输出：

- `technical_indicators`

## 3. MVP 指标清单

必须实现：

- MA5、MA10、MA20、MA60、MA120。
- BOLL：中轨、上轨、下轨、带宽、带宽变化。
- RSI14。
- ATR14。
- volume_ma20。
- volume_ratio_20。

第二阶段：

- MACD。
- KDJ。
- OBV。
- 涨停、炸板、连板等 A 股特征指标。

## 4. 均线计算

```text
MA_N = close rolling mean over N trading days
```

注意：

- 前 N-1 个交易日结果为空。
- 默认使用前复权 close。
- 均线只提供数值，不在本层判断多头排列。

## 5. BOLL 计算

默认参数：

```text
计算方式参考：
N:=20;
P:=2;

{ 中轨计算：使用 SMA 平滑移动平均线，这是精确匹配的关键 }
MID := SMA(CLOSE, N, 1);

{ 标准差计算：使用 STD 函数，通用方法 }
VART1:=POW((CLOSE-MID),2);
VART2:=MA(VART1,N);
VART3:=SQRT(VART2);

{ 上下轨计算 }
UPPER := MID + P * VART3;
LOWER := MID - P * VART3;
```

注意：

- middle 为 0 或 null 时 width 为空。
- 前 19 个交易日无法计算 BOLL。
- 参数必须写入版本或配置，后续变更可追溯。

## 6. 成交量指标

```text
volume_ma20 = MA(volume, 20)
volume_ratio_20 = volume / volume_ma20
```

缩量、放量判断不在指标层完成，只提供量能倍率。

## 7. ATR 计算

```text
TR = max(
  high - low,
  abs(high - previous_close),
  abs(low - previous_close)
)
ATR14 = MA(TR, 14)
```

ATR 用于衡量波动率和回踩幅度是否温和。

## 8. RSI 计算

默认参数：

```text
N = 14
RSI = avg_gain / (avg_gain + avg_loss) * 100
```

RSI 第一阶段仅作为特征输入，不作为硬过滤。

## 9. 版本管理

示例：

```text
ind_v1.0.0:
  ma: simple moving average
  boll: close, 20, 2 std
  rsi: 14
  atr: 14
```

指标算法或参数变化时升级版本。

## 10. 推荐实现

指标计算应支持批量模式：

```python
def calculate_indicators(bars: DataFrame, version: str) -> DataFrame:
    ...
```

不要在循环中逐行访问数据库。推荐一次拉取窗口数据，在内存中向量化计算后批量写入。

## 11. 测试要求

必须提供：

- 小样本手工校验。
- 与 pandas-ta 或 TA-Lib 对比校验。
- 边界测试：数据不足、停牌、价格为 0、成交量为 0。

## 12. 验收标准

- 指标结果可重复。
- 同一版本同一输入输出一致。
- 数据不足时输出 null，而不是错误值。
- 指标表可以支撑后续特征工程查询。
